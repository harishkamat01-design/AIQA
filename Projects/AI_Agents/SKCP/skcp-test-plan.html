Test Strategy Overview
Guiding principles, philosophy, and approach

2.1 Testing Philosophy

The SKCP B2B Dashboard is a single self-contained HTML file with zero build steps. All logic, UI, charts, forms, and multilingual content are rendered in-browser using vanilla JavaScript. The testing strategy therefore focuses on browser-based functional validation, in-memory data flow, and cross-cutting concerns (theme, language, responsive layout).

2.2 Testing Pyramid

LevelTypeVolumeApproachTool
🔺 TopEnd-to-End / UATLow (12 flows)Full user journey from login to payment receiptManual / Browser
🔶 MidIntegrationMedium (35 cases)Cross-module data flow (Quote→Order→Dispatch→Payment)Manual / Browser DevTools
🟩 BaseFunctional / UnitHigh (100 cases)Per-module feature validationManual / Browser Console

2.3 Test Prioritisation (MoSCoW)

PriorityModules / FeaturesRationale
MUST TestPayment Centre, Quote Builder, Labour Check-in, Wages Calculation, Dispatch 6-step, Language SwitcherBusiness-critical; financial and operational impact if broken
SHOULD TestOrders, Inventory, Production, Timesheet, Calendar, Pipeline, Online OrdersCore daily-use modules; defects impact productivity
COULD TestAbout Us, Contact, Map, Reports export, Settings togglesInformational/admin; lower business risk

Impact: Critical | Probability: Low | Module: Wages, Quote Builder
Mitigation: Dedicated S1 test cases (TC-WAG-01, TC-QB-01/02); independent verification by Accounts manager during UAT
HIGH
RISK-02: CDN unavailability (Chart.js, Tabler Icons, Google Fonts)
Impact: High | Probability: Very Low | Module: All sections
Mitigation: Test environment has active internet; version-pinned CDN URLs reduce breaking change risk; document CDN SLA
HIGH
RISK-03: Chart.js canvas rendering fails on Safari / Firefox
Impact: Medium | Probability: Low | Module: Overview, Production, Reports, Pipeline
Mitigation: TC-PERF-01 executed on all 4 browsers; Chart.js 4.4.1 has broad support; fallback aria-labels ensure content accessible
MEDIUM
RISK-04: Language switch breaks nav-badge counts (badges lost on text update)
Impact: Medium | Probability: Medium | Module: Sidebar, all languages
Mitigation: TC-LANG-01 step 6 specifically verifies badge preservation; setLang() appends badge after text node update
MEDIUM
RISK-05: Timesheet fails to exclude all holidays correctly
Impact: Medium | Probability: Low | Module: Timesheet, Wages calculation
Mitigation: TC-TS-01 cross-references HOL object; single source of truth pattern reduces inconsistency risk
MEDIUM
RISK-06: In-memory data lost on page refresh (expected but may confuse UAT users)
Impact: Low | Probability: Certain | Module: All (by design)

LOW
