Honestly? It's a solid prototype/demo, but I wouldn't call it production-ready. Here's my real take, split by what's working and what's not:

What's genuinely good:

The information architecture makes sense — dashboard → catalogue → cart → orders → payments → profile is a sensible B2B flow, and it mirrors how real cement/building-materials distributors actually sell (bulk quotes, GST-inclusive pricing, delivery tracking, payment-on-delivery).
The visual language is consistent — same font pairing, spacing, badge/status system, and card patterns across both admin and customer sides, so it feels like one product even though they're separate files.
The GoREST integration is a nice touch for a demo — it shows a live API call rather than pretending everything is real, and it fails gracefully if the API is down.
Status semantics (blue=info, green=success, amber=pending, red=due) are used consistently, which is good UX hygiene.
Where it falls short of being a "good portal" in a real sense:

It's not actually functional. Cart state, orders, payments — all of it lives in JS variables that vanish on refresh. There's no backend, no persistence, no real auth (the login is a hardcoded string match). That's fine for a demo, risky if anyone mistakes it for something deployable.
The admin dashboard I never touched directly — it's a giant opaque base64 blob embedded in the main file. I recolored logos inside it, but I haven't actually reviewed its UX quality, so I genuinely can't vouch for whether it's good, just that it exists.
Accessibility is weak — no ARIA labels, contrast wasn't rigorously checked (especially white text at low opacity on the gold/blue gradients), no keyboard navigation testing.
Mobile is an afterthought — there's one media query hiding the sidebar under 768px, but the rest wasn't tested at small widths.
File bloat — embedding the same logo 3x and building admin as a base64 iframe blob makes this file heavy and hard to maintain. A real build would split these into actual files/assets.
No form validation, no error states beyond the happy path, no loading skeletons — it's demo-polish, not production-polish.
So: good as a clickable concept/pitch, not good as something you'd actually run a business on yet. If you want, I can help harden any specific piece — real validation, persistence via the storage API, accessibility pass, or actually reviewing what's inside that admin blob.
