How the Advanced RAG was built

This explainer captures the architecture behind the VWO test-case assistant: ingest, chunk, embed, retrieve, rerank, and answer.

Pipeline

1. Ingest
CSV/XLSX rows become structured documents.
2. Chunk
Each row becomes a searchable chunk.
3. Embed
Dense + sparse features build the retrieval index.
4. Rerank
Hybrid scores are fused and reranked.
5. Generate
GROQ produces a grounded answer with citations.

Core components

  • CSV ingestion and schema normalization
  • Chunking pipeline with overlap control
  • Hybrid search using dense and sparse signals
  • RRF fusion for candidate ranking
  • GROQ rewrite + grounded generation

What the UI shows

  • Live pipeline tracker
  • Interactive chat experience
  • Snippet-based evidence with chunk citations
  • Chunk viewer for search and filtering
